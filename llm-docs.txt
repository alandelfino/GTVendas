# GT Vendas — API Mobile: Referência Completa
Versão: 2.0 | Base URL: https://gtvendas.grupotitaniumjeans.com.br
Escopo: todos os endpoints acessíveis pelo aplicativo mobile via JWT Bearer Token

---

## AUTENTICAÇÃO JWT

O app mobile usa autenticação via JWT Bearer Token.
Todos os endpoints (exceto /api/mobile/login e /api/mobile/refresh) exigem o header:
  Authorization: Bearer <accessToken>

O usuário autenticado sempre tem role "user" e está vinculado a um representanteId.
O backend filtra automaticamente os dados pelo representanteId do token — não é necessário enviá-lo nas queries.

Fluxo completo:
1. POST /api/mobile/login → recebe accessToken (15 min) + refreshToken (30 dias)
2. Armazenar accessToken e refreshToken em SecureStorage
3. Enviar "Authorization: Bearer <accessToken>" em toda requisição autenticada
4. Ao receber HTTP 401: chamar POST /api/mobile/refresh → novo accessToken
5. Reenviar a requisição original com o novo accessToken

## MÓDULO 1: AUTENTICAÇÃO

### POST /api/mobile/login
Autentica o usuário e retorna tokens JWT.
Auth: nenhuma (público)
Content-Type: application/json

Entrada (Body JSON):
  username: string (obrigatório) — nome de usuário
  password: string (obrigatório) — senha

Saída 200 (JSON):
  accessToken: string — JWT válido por 15 minutos
  refreshToken: string — JWT válido por 30 dias, armazenar em SecureStorage
  user:
    id: string (UUID) — identificador único do usuário no sistema
    username: string — nome de usuário
    role: string — sempre "user" para representantes
    representanteId: string | null — ID externo do representante vinculado
    nomeCompleto: string | null — nome completo do usuário
    companiesIds: string[] — lista de IDs de empresas às quais o representante tem acesso

Erros:
  400: { "error": "username e password são obrigatórios" }
  401: { "error": "Credenciais inválidas" }

---

### POST /api/mobile/refresh
Renova o accessToken usando o refreshToken.
Auth: nenhuma — o refreshToken é a credencial
Content-Type: application/json

Entrada (Body JSON):
  refreshToken: string (obrigatório) — token salvo no SecureStorage

Saída 200 (JSON):
  accessToken: string — novo JWT válido por 15 minutos

Erros:
  400: { "error": "refreshToken é obrigatório" }
  401: { "error": "Token inválido ou expirado" }

---

### GET /api/mobile/me
Retorna os dados do usuário autenticado pelo JWT.
Auth: Bearer Token (obrigatório)

Entrada: nenhuma

Saída 200 (JSON):
  id: string (UUID)
  username: string
  role: string
  representanteId: string | null
  nomeCompleto: string | null
  companiesIds: string[]

Erros:
  401: { "error": "Nao autenticado" }

## MÓDULO 3: CLIENTES

### GET /api/erp/clientes
Lista os clientes da carteira do representante autenticado com paginação.
Auth: Bearer Token (obrigatório)
Nota: role "user" vê automaticamente apenas os clientes do seu representanteId

Entrada (Query Params):
  search: string (opcional) — filtro por nome ou CNPJ
  page: number (opcional, default 1) — página atual
  limit: number (opcional, default 30, máx 200) — itens por página
  status: string (opcional) — "ativo" | "inativo" | "restrito" | "bloqueado" | "semAtividade"

Saída 200 (JSON):
  data: array de Cliente
    idExterno: string — ID único do cliente no ERP
    criadoEm: number — timestamp Unix em milissegundos
    modificadoEm: number — timestamp Unix em milissegundos
    nome: string — razão social do cliente
    fantasia: string | null — nome fantasia
    cnpj: string | null — CNPJ do cliente
    statusDescricao: string — "Ativo" | "Inativo" | "Restrição" | "Bloqueado"
    representanteId: string — ID externo do representante responsável
    endereco:
      cep: string | null
      logradouro: string | null
      numero: number | null
      bairro: string | null
      cidade: string | null
      uf: string | null — sigla do estado (2 letras)
      pais: string | null
  total: number — total de registros encontrados
  page: number — página atual
  limit: number — itens por página

---

### GET /api/erp/clientes/stats
Retorna estatísticas resumidas dos clientes do representante.
Auth: Bearer Token (obrigatório)

Entrada: nenhuma

Saída 200 (JSON):

  ativos: number — clientes com status "Ativo"
  inativos: number — clientes com status "Inativo"
  restritos: number — clientes com status "Restrição"

    cadastradoEm: number — timestamp Unix em milissegundos
    valorTotal: number — valor total em CENTAVOS (ex: 1990 = R$19,90)
    status: string — "Pendente" | "Concluído" | "Expedido" | "Faturado" | "Cancelado"
    cliente:
      idExterno: string — ID do cliente
      nome: string — razão social
      fantasia: string | null — nome fantasia
      cnpj: string | null
      cidade: string | null
      uf: string | null
      statusDescricao: string | null
  total: number
  page: number
  limit: number

---

### GET /api/erp/pedidos/:idExterno
Retorna um pedido completo com todos os seus itens.
Auth: Bearer Token (obrigatório)

Entrada (Path Params):
  idExterno: string (obrigatório) — ID externo do pedido

Saída 200 (JSON):
  idExterno: string
  cadastradoEm: number — timestamp em ms
  valorTotal: number — em centavos
  status: string
  cliente: objeto Cliente (idExterno, nome, fantasia, cnpj, cidade, uf, statusDescricao)
  descontos: array de { tipo: string, valor: number } — valor em centavos
  itens: array de ItemPedido
    idExterno: string — ID do produto
    nome: string — nome do produto
    precoVendido: number — preço unitário em centavos
    grade: array de { nome: string, quantidade: number } — tamanhos e quantidades

Erros:
  404: { "error": "Pedido não encontrado" }

## MÓDULO 13: ASSISTENTE GT (CHAT IA)

O Assistente GT é um chatbot com GPT-4 e Function Calling.
Comunicação via HTTP REST síncrono — await até a resposta completa.

### POST /api/mobile/chat/message
Envia uma mensagem ao assistente e aguarda a resposta completa.
Auth: Bearer Token (obrigatório) | Role: user com representanteId
Content-Type: application/json

Entrada (Body JSON):
  content: string (obrigatório) — texto da mensagem do usuário
  sessionId: number (obrigatório) — ID da sessão de chat (deve ser criada previamente)
  voiceMessageId: number (opcional) — ID de uma mensagem de voz gravada previamente

Saída 200 (JSON):
  messageId: number — ID da mensagem de resposta salva no banco
  content: string — texto completo da resposta do assistente
  events: array de eventos gerados durante o processamento
    Evento tool_call: { "type": "tool_call", "tool": "nome_da_funcao" }
    Evento pdf: { "type": "pdf", "reportId": "string (UUID)", "url": "string", "titulo": "string" }
    Evento map_embed: { "type": "map_embed", "embedUrl": "string", "mapsUrl": "string" }
    Evento html_widget: { "type": "html_widget", "widgetId": "string", "html": "string", "titulo": "string" }

Erros:
  400: { "error": "content é obrigatório e não pode ser vazio" }
  400: { "error": "sessionId é obrigatório e deve ser um inteiro positivo" }
  403: { "error": "Sessão não encontrada ou sem permissão" }
  500: { "error": "mensagem de erro interno" }

---

### GET /api/rep/chat/sessions/:id/messages
Retorna o histórico de mensagens de uma sessão.
Auth: Bearer Token (obrigatório) | Role: user com representanteId

Entrada (Path Params):
  id: number (obrigatório) — ID inteiro da sessão

Saída 200 (JSON Array de ChatMessage):
  id: number — ID da mensagem

  content: string — texto da mensagem (em mensagens de voz: transcrição do áudio)
  criadoEm: string — ISO 8601
  hasAudio: boolean — true se o usuário enviou áudio gravado (habilitar player)
  hasTts: boolean — true se o TTS da resposta já está em cache (habilitar reprodução)
  transcricao: string | null — transcrição do áudio (igual a content para mensagens de voz)
  audioMime: string | null — MIME type do áudio gravado ("audio/webm", "audio/mp4", "audio/mpeg")

Nota: os binários de áudio (audio_data, tts_data) NÃO são retornados aqui.

## CÓDIGOS DE ERRO

400 Bad Request — dados inválidos ou campos obrigatórios ausentes
  { "error": "mensagem descritiva" }
  ou { "success": false, "error": "mensagem", "campos": { campo: "motivo" } }

401 Unauthorized — não autenticado ou token expirado
  { "error": "Nao autenticado" }
  Ação: chamar POST /api/mobile/refresh com o refreshToken

403 Forbidden — autenticado mas sem permissão para o recurso
  { "error": "Acesso negado" }

404 Not Found — recurso não encontrado
  { "error": "mensagem" }

409 Conflict — recurso duplicado (idExterno já existe)
  { "success": false, "error": "idExterno '...' ja existe" }

500 Internal Server Error — erro interno
  { "error": "mensagem técnica" }

## CONVENÇÕES IMPORTANTES

- Valores monetários: todos em CENTAVOS (integer). Dividir por 100 para exibir em reais.
  Exemplo: 1990 = R$ 19,90 | 815950 = R$ 8.159,50
- Metas de venda: em PEÇAS (unidades), NÃO em reais. Nunca confundir.
- Timestamps: Unix timestamp em milissegundos (Number).
  Converter: new Date(timestamp).toLocaleDateString("pt-BR")
- Datas ISO: formato "YYYY-MM-DD" para datas, "YYYY-MM-DDTHH:mm:ss.sssZ" para datetime
- Paginação: page começa em 1; a resposta inclui total de registros
- O representanteId do usuário autenticado é aplicado automaticamente pelo backend —
  nunca é necessário enviá-lo como query param quando autenticado com JWT Bearer

## FLUXO RECOMENDADO PARA O APP MOBILE

1. Login: POST /api/mobile/login → salvar accessToken + refreshToken em SecureStorage
2. Toda requisição: header "Authorization: Bearer <accessToken>"
3. Ao receber 401: POST /api/mobile/refresh → novo accessToken → reenviar requisição
4. Ao iniciar tela do chat:
   a. POST /api/rep/chat/sessions → obter sessionId
   b. GET /api/rep/chat/sessions/:id/messages → carregar histórico
   c. POST /api/mobile/chat/message { content, sessionId } → enviar mensagem
   d. Exibir data.content como resposta do assistente
   e. Se data.events contiver "pdf" ou "html_widget": oferecer visualização do relatório
5. Para voz:
   a. Gravar áudio e transcrever: POST /api/rep/chat/transcribe → obter text
   b. Enviar transcrição: POST /api/mobile/chat/message { content: text, sessionId }
   c. Para TTS da resposta: POST /api/rep/chat/messages/:id/tts → reproduzir áudio
6. Para permissões de microfone no dispositivo:
   Android: RECORD_AUDIO no AndroidManifest.xml
   iOS: NSMicrophoneUsageDescription no Info.plist
